===== To make a Custom Response rules =====

/-- usage --/
creating a custom response lets you make your own dynamic bot messages,
you can use it to provide automated responses such as faq or product details
by making a custom response you can make a dynamic responses according to what response you want to give to the user if they need something

/-- creating a cusom response --/
inside the folder [\Chatbot\Responses\], you can make you own custom responses
all custom responses are only detected here, otherwise it wont work


/-- child class --/
you must extend this class from [Chatbot\Core\Response] to inherit the parent's functions such as getResponse, otherwise it wont work
[class CustomResponse exteds Response] -- a child class of Response


/-- making own rules --/
make an array variable called [$rules] to the class
[protected array $rules = ['keyword'=>'response']]
once the user send a message containing the keyword, it will response according to the value you have given


/-- trigger word --/
adding trigger word is essential to the class letting the bot know how it filters the responses
trigger words are the identifier of the response
you can write [public static string $triggerWord = 'greeting';] to put
custom  responses without trigger word are most likely to get ignored by the handler so to be sure, you must right it


/-- error response --/
error response are the throwable response if the message is not in the rules
the [getResopnse()] method automatically throw an error
error response are optional,
if error reponse is not existing or is null, then the bot will auto fill the response from [\Chatbot\Core\ErrorResponses::class]
to make an error response simply type [protected array $errorResponse = [];]

===== Restrictors =====

/-- usage --/
restrictors are the middleware of you custom responses
it detects what are allowed to gain the access of the information of the rules you have middleware
for example, only admin can know how many users are logged today, then restrictors blocks that information of user if auth is not admin


/-- creating a cusom restrictors --/
inside the folder [\Chatbot\Restrictors\], you can make you own custom restrictors


/-- setup --/
extend the restrictor to the parent [\Chatbot\Core\BaseRestrictor]
[class TestRestrictor extends BaseRestrictor]

--check if allowed--
[
    public function passes(): bool

/-- implements in responses --/
you can activate restrictions by implementing restrictions using
[class TestResponse extends Response implements HasRestriction]
it lets the handler to know that it has restriction before making a response

-- register restrictor/s --
to register restrictors on you responses just use this function
[
    public function getRestrictor(): string|array
    {
        return TestRestrictor::class;
        // or multiple: return [TestRestrictor::class, AnotherRestrictor::class];
    }
]
by that it automatically restricts all the information given by your bot
